# Implementation Plan - Map Search, Results Plotting, and Details Panel

This plan details the design and implementation of the Map search and visualization features. It allows users to search for tree types, plot locations on an OpenStreetMap map, view a list of matching results, and view detailed information for any selected tree.

## User Review Required

> [!IMPORTANT]
> **Duplicate IDs in HTML**: There is currently a duplicate `id="treetypeinput"` on the distance input field in [index.html](file:///c:/Users/ebb0039/ethan-balon.github.io/TreeSpotter/index.html). We will change the distance input to `id="distanceinput"` to resolve this and ensure HTML validity.
>
> **Sub-section Navigation in Map Sidebar**: We will toggle the `.active` class between `#map_search_section`, `#map_results_section`, and `#map_details_section` inside the aside panel to swap views.
> - To support smooth back-and-forth flow, we will add back buttons: "Back to Search" inside the results view and "Back to Results" inside the details view.

## Open Questions

> [!NOTE]
> **Map Center on Results**: When plotting search results, should the map automatically pan and adjust its zoom level to fit the bounds of all plotted tree markers?
> * **Proposed Design**: Yes, we will compute the Leaflet LatLngBounds of all matching trees and call `map.fitBounds(bounds)` so the user gets an immediate view of the results. If a single tree is returned, we will center directly on it.

### TreeSpotter Views

---

#### [NEW] [mapView.js](file:///c:/Users/ebb0039/ethan-balon.github.io/TreeSpotter/views/mapView.js)
- Implement `MapView` class to manage all DOM and Leaflet interactions:
  - Cache elements: `#treetypediv`, `#treetypeinput`, `#map_search_section`, `#map_results_section`, `#map_details_section`, `#map_results_list`, `#results_back_button`, `#tree_details_container`, and `#details_back_button`.
  - Implement `initMap(lat, lng, zoom)` to initialize and return the Leaflet map object.
  - Implement navigation toggle methods: `showSearch()`, `showResults()`, and `showDetails()`.
  - Implement `renderResults(trees, onTreeClick)` to dynamically generate list elements under `#map_results_list`.
  - Implement `renderDetails(tree)` to populate HTML details inside `#tree_details_container`.
  - Implement `plotMarkers(trees, map, onMarkerClick)` to display markers on the map, return reference to markers, and auto-fit map view bounds.
  - Implement `clearMarkers(markers, map)` to clean old markers from the map before plotting new results.

### TreeSpotter Controllers

---

#### [NEW] [mapController.js](file:///c:/Users/ebb0039/ethan-balon.github.io/TreeSpotter/controllers/mapController.js)
- Implement `MapController` class to coordinate state, searches, and clicks:
  - Instantiate `MapView` and `DataController`.
  - Initialize the Leaflet map and track current active markers.
  - Bind form submission on the search form:
    1. Fetch trees matching search input from `DataController.fetchTrees(query)`.
    2. Render search results list and plot markers on the map.
    3. Transition view state to results panel (`showResults()`).
  - Bind item clicks (on map markers and result list items) to transition to details view (`showDetails()`) and render details of the selected tree.
  - Bind navigation buttons to transition back to search or results views.

#### [MODIFY] [sectionController.js](file:///c:/Users/ebb0039/ethan-balon.github.io/TreeSpotter/controllers/sectionController.js)
